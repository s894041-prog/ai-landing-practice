CHAPTER 5

踩過的坑，幫你省時間

01
Session 用很快的原因
貼了太多文件、長截圖、反覆修改——Token 消耗比你想像的快 3 倍
02
MCP 到 Chrome 的資源消耗
Chrome extension + MCP server 同時運行，記憶體佔用需要關注
03
學務系統只能用 find+form_input
政府系統多為舊式 DOM，navigate/click 常失敗，改用 find 定位元素才穩定
04
正確率八成——何時需要人工確認？
成績、公文、金額類資料——AI 輸出永遠需要人眼最終確認
05
Token / Session / Limit 怎麼控制
善用 /compact、拆分任務、用 Haiku 處理簡單工作，有效延長 Session
06
不要追求「一次到位」
先跑通 80%，再用人工補 20%——半自動就是最好的自動化
CHAPTER 6

建立你自己的知識庫

為什麼要用 MD 檔案？
Markdown 是最輕量的結構化格式，Claude 對 MD 的解讀能力遠優於 Word/PDF——讓 AI 更容易理解你的內容
Skill 是什麼，怎麼包？
把重複性任務的「作法 + 範例 + 規則」寫成一份 SKILL.md，Claude 每次執行都會遵循——等於在訓練你的私人助理
個人文章知識庫的建法
用 CLAUDE.md 記錄個人偏好、常用格式、學校規定——讓 Claude 每次對話都「認識你」
讓 Claude 認識你這個人
Memory 功能 + 個人化 CLAUDE.md：記錄你的說話風格、慣用詞彙、工作情境，對話品質直接提升
CHAPTER 7

我對 AI 的真心話

不是每件事都適合 AI
陪孩子的時間、需要同理心的對話——這些是人類的專屬領域
「把認知資源留給真正需要人類判斷的事」
——讓 AI 做搬運工，讓老師做教育者
缺的不是工具
缺的是肯陪孩子走的老師。AI 是手段，陪伴才是目的。
半自動就夠了
不用追求全自動。80% AI + 20% 人工確認，才是最穩定的工作流。
